Conceptos Básicos de Redes

La idea básica de las redes es conectar dispositivos para compartir datos y recursos. Las redes pueden variar desde redes locales pequeñas hasta grandes redes globales como internet.

Tipos de Redes

Las redes se clasifican según su tamaño y alcance:

  • LAN (Red de Área Local): Cubre un área geográfica pequeña, como un solo edificio.
  • WAN (Red de Área Amplia): Cubre un área geográfica grande, a menudo a través de ciudades o países.
  • MAN (Red de Área Metropolitana): Cubre una ciudad o un gran campus.

Topologías de Red

La disposición de una red, conocida como topología, puede incluir:

>
  • Estrella: Todos los dispositivos se conectan a un concentrador o conmutador central.
  • Bus: Todos los dispositivos se conectan a un cable central único.
  • Anillo: Los dispositivos están conectados en una disposición circular.
  • Malla: Los dispositivos están interconectados, proporcionando múltiples rutas para los datos.
  • Híbrida: Una combinación de dos o más topologías.

Direccionamiento IP

Cada dispositivo en una red recibe una dirección IP única. Esta dirección se utiliza para identificar y comunicarse con los dispositivos en la red. Las direcciones IP pueden ser IPv4 o IPv6.

>

Subnetting

Divide una red grande en segmentos o subredes más pequeños y manejables. Esto mejora el rendimiento y la seguridad.

VLAN (Red de Área Local Virtual)

Una VLAN permite segmentar una red física en varias redes lógicas. Los dispositivos en la misma VLAN pueden comunicarse como si estuvieran en la misma red física, incluso si no lo están.

Protocolos de Red

Reglas y convenciones para la comunicación entre dispositivos de red. Protocolos comunes incluyen:

  • TCP/IP: La suite de protocolos fundamental para la comunicación en internet.
  • HTTP/HTTPS: Protocolos utilizados para la transferencia de páginas web.
  • FTP: Protocolo para la transferencia de archivos.
  • DNS: Sistema de Nombres de Dominio para resolver nombres de dominio en direcciones IP.

Modelos de Redes

Los modelos de redes definen cómo operan las redes y ayudan en el diseño y comprensión de las redes al dividirlas en capas manejables. Cada modelo proporciona un marco para entender cómo interactúan los diferentes protocolos de red.

Modelo OSI

El modelo OSI estandariza las funciones de una red en siete capas. A continuación se muestra un diagrama que ilustra estas capas:

>
  • Física: Se ocupa de la conexión física entre dispositivos, como cables e interfaces de hardware. Define los aspectos eléctricos, mecánicos y procedimentales necesarios para la transmisión de datos.
  • Enlace de Datos: Gestiona la transferencia de datos de nodo a nodo y la detección de errores. Asegura que los datos enviados desde la capa Física sean recibidos correctamente y maneja la corrección de errores a este nivel.
  • Red: Maneja el reenvío de paquetes y el enrutamiento. Esta capa es responsable de determinar el mejor camino para que los paquetes de datos viajen desde la fuente hasta el destino a través de diferentes redes.
  • Transporte: Asegura la comunicación de extremo a extremo y la recuperación de errores. Proporciona una transferencia de datos fiable estableciendo conexiones, gestionando el control de flujo y detectando errores.
  • Sesión: Gestiona las sesiones entre aplicaciones. Controla el diálogo entre dos sistemas, estableciendo, manteniendo y terminando conexiones según sea necesario.
  • Presentación: Traduce los datos a un formato adecuado para la aplicación. Esta capa maneja la cifrado, compresión y traducción de datos entre diferentes formatos de datos.
  • Aplicación: Proporciona servicios de red a las aplicaciones. Es la interfaz donde las aplicaciones de usuario final interactúan con la red, incluyendo servicios como correo electrónico, transferencia de archivos y navegación web.

    • Ultima capa 8 el usuario final o CUSTOMER: Aunque el modelo OSI tiene siete capas, algunos dicen que la capa 8 es el "usuario final", ¡donde ocurren todos los errores de red! La capa 8 es conocida por ser responsable de muchos problemas en la red debido a errores humanos.

Modelo TCP/IP

El modelo TCP/IP es una simplificación del modelo OSI y se utiliza en redes modernas. A continuación se muestra un diagrama que ilustra las cuatro capas del modelo TCP/IP:

>
  • Enlace: Abarca las funcionalidades de las capas de Enlace de Datos y Física del OSI. Maneja el hardware y los protocolos necesarios para la transmisión de datos a través de un segmento de red.
  • Internet: Equivalente a la capa de Red del OSI. Se encarga del direccionamiento lógico, enrutamiento y reenvío de paquetes a través de límites de red.
  • Transporte: Coincide con la capa de Transporte del OSI. Proporciona servicios de comunicación de extremo a extremo para aplicaciones, asegurando que los datos se transfieran de manera fiable y en el orden correcto.
  • Aplicación: Combina las capas de Sesión, Presentación y Aplicación del OSI. Soporta diversos servicios y protocolos de red, como HTTP, FTP y SMTP, permitiendo que las aplicaciones se comuniquen a través de la red.

Topologías de Red

Las topologías de red definen cómo se conectan y comunican los dispositivos dentro de una red. A continuación se muestra un diagrama que ilustra algunas de las topologías más comunes:

>
  • Estrella: Todos los dispositivos se conectan a un hub o switch central. Facilita la gestión y expansión de la red.
  • Bus: Todos los dispositivos se conectan a un único cable central. Es simple y económico, pero puede tener problemas de rendimiento con muchas estaciones.
  • Anillo: Los dispositivos están conectados en una configuración circular. Los datos viajan en una sola dirección o en ambas direcciones para evitar fallos.

Nota humorística: Aunque el modelo OSI tiene siete capas, algunos dicen que la capa 8 es el "usuario final", ¡donde ocurren todos los errores de red! La capa 8 es conocida por ser responsable de muchos problemas en la red debido a errores humanos.

Dispositivos de Red

Los dispositivos de red son componentes de hardware esenciales que ayudan a conectar y gestionar redes. Cada dispositivo cumple un papel específico en las operaciones de red. A continuación, se detalla la función de cada dispositivo, su participación en el modelo OSI, las topologías en las que se utilizan, sus ventajas y desventajas, así como advertencias de manipulación incorrecta y recomendaciones para su instalación y manejo:

  • Ruteadores o Routers:

    Función: Dirigen los paquetes de datos entre diferentes redes y gestionan el tráfico de red.

    Modelo OSI: Operan en la Capa 3 (Capa de Red).

    Topologías: Se utilizan en topologías de estrella, malla y árbol para conectar múltiples redes.

    Ventajas: Permiten la interconexión de redes, gestionan el tráfico eficientemente y proporcionan funcionalidades avanzadas como NAT y VPN.

    Desventajas: Pueden ser costosos y requieren configuración y mantenimiento especializados.

    Advertencias: Manipular incorrectamente la configuración de un router puede causar pérdida de conectividad entre redes, problemas de seguridad y reducción en el rendimiento de la red.

    Instalación y Manipulación: Solo deben ser instalados y configurados por personal capacitado en redes, como administradores de red o ingenieros de redes.

    >
  • Conmutadores o Switches:

    Función: Conectan dispositivos dentro de una sola red y gestionan el tráfico de datos en la capa de enlace de datos.

    Modelo OSI: Operan en la Capa 2 (Capa de Enlace de Datos).

    Topologías: Utilizados en topologías de estrella y malla para conectar dispositivos en una red local.

    Ventajas: Mejoran la eficiencia de la red al reducir las colisiones y el tráfico innecesario, y permiten la segmentación de redes.

    Desventajas: No manejan el tráfico entre diferentes redes y pueden ser costosos en redes grandes.

    Advertencias: Una configuración incorrecta puede causar cuellos de botella en la red, problemas de comunicación entre dispositivos y pérdida de datos.

    Instalación y Manipulación: Deben ser instalados y configurados por personal con experiencia en redes, como técnicos de redes o ingenieros de redes.

    >
  • Concentradores o Hubs:

    Función: Conectan múltiples dispositivos en una red, transmitiendo datos a todos los dispositivos conectados.

    Modelo OSI: Operan en la Capa 1 (Capa Física).

    Topologías: Usados en topologías de estrella y bus para conectar dispositivos.

    Ventajas: Económicos y simples de configurar.

    Desventajas: No gestionan el tráfico de datos, lo que puede causar colisiones y congestión en la red.

    Advertencias: El uso inadecuado puede causar un rendimiento deficiente de la red y aumentar las colisiones de datos.

    Instalación y Manipulación: Pueden ser instalados por personal con conocimientos básicos en redes, pero es recomendable que el mantenimiento sea realizado por técnicos de redes para evitar problemas.

    >
  • Modems:

    Función: Convierten datos digitales en señales analógicas para transmisión y viceversa.

    Modelo OSI: Operan en la Capa 1 (Capa Física) y Capa 2 (Capa de Enlace de Datos).

    Topologías: Se utilizan en topologías punto a punto para conectar redes a través de líneas telefónicas o de cable.

    Ventajas: Permiten la conexión a Internet a través de diversos tipos de líneas de comunicación.

    Desventajas: Pueden ser lentos en comparación con las tecnologías modernas y pueden requerir configuración adicional.

    Advertencias: Una configuración incorrecta puede resultar en una conexión a Internet inestable o lenta.

    Instalación y Manipulación: Los modems deben ser instalados y configurados por técnicos de soporte o personal capacitado en redes y comunicaciones.

    >
  • Portátiles:

    Función: Computadoras portátiles utilizadas para diversas tareas, actuando como receptores y emisores de datos.

    Modelo OSI: Participan en la Capa 1 (Capa Física), Capa 2 (Capa de Enlace de Datos), y Capa 3 (Capa de Red) dependiendo del tipo de conexión.

    Topologías: Se utilizan en cualquier topología de red donde los usuarios necesiten movilidad y acceso a la red.

    Ventajas: Portátiles y versátiles, permiten la conexión a redes inalámbricas y cableadas.

    Desventajas: Dependen de la conexión de red para acceder a los recursos de la red, y pueden tener limitaciones de rendimiento comparadas con estaciones de trabajo de escritorio.

    Advertencias: El mal uso o la falta de mantenimiento pueden afectar la conectividad y el rendimiento de la red.

    Instalación y Manipulación: Los usuarios finales pueden instalar y usar portátiles, pero el soporte técnico debe encargarse de la configuración avanzada y la resolución de problemas.

    >
  • Puntos de Acceso:

    Función: Permiten que los dispositivos inalámbricos se conecten a una red cableada a través de Wi-Fi.

    Modelo OSI: Operan en la Capa 1 (Capa Física) y Capa 2 (Capa de Enlace de Datos).

    Topologías: Utilizados en topologías de estrella para conectar dispositivos inalámbricos a una red cableada.

    Ventajas: Facilitan la conexión inalámbrica, extendiendo el alcance de la red y proporcionando acceso en áreas donde las conexiones cableadas no son prácticas.

    Desventajas: La cobertura y el rendimiento pueden verse afectados por la interferencia y la distancia.

    Advertencias: La mala configuración o la falta de seguridad pueden exponer la red a riesgos y reducir la calidad de la conexión.

    Instalación y Manipulación: Deben ser instalados y configurados por personal capacitado en redes para garantizar una cobertura adecuada y seguridad.

    >
  • Repetidores:

    Función: Amplifican la señal para extender la distancia de transmisión en una red.

    Modelo OSI: Operan en la Capa 1 (Capa Física).

    Topologías: Utilizados en topologías de bus, estrella y malla para extender el alcance de la red.

    Ventajas: Extienden la distancia de la red y mejoran la cobertura en áreas de difícil acceso.

    Desventajas: Pueden introducir latencia y no solucionan problemas de interferencia en la señal.

    Advertencias: Un uso inadecuado puede causar una reducción en el rendimiento de la red y aumentar la latencia.

    Instalación y Manipulación: La instalación y configuración deben ser realizadas por técnicos para asegurar que la señal se amplifique correctamente y sin interferencias.

    >

Direcciones IP y Subnetting

Las direcciones IP son identificadores únicos asignados a cada dispositivo en una red. Sin direcciones IP, los dispositivos no podrían encontrarse ni comunicarse entre sí. Una correcta gestión de las direcciones IP asegura que la red funcione de manera eficiente y segura. Exploraremos a fondo cómo funcionan las direcciones IP, las máscaras de subred, y cómo se asignan las direcciones IP.

Estructura de las Direcciones IP

Las direcciones IP son como las direcciones en una ciudad: indican dónde se encuentra un dispositivo dentro de una red. Hay dos versiones principales:

  • IPv4 (Internet Protocol version 4): Esta versión utiliza una dirección de 32 bits, dividida en cuatro octetos de 8 bits cada uno. Estos octetos se representan en formato decimal, separados por puntos. Por ejemplo, 192.168.1.1. Esta estructura permite aproximadamente 4.3 mil millones de direcciones únicas, suficientes para la mayoría de las redes privadas y de pequeña a mediana escala.
  • IPv6 (Internet Protocol version 6): Debido al crecimiento exponencial de dispositivos conectados a Internet, se desarrolló IPv6, que utiliza una dirección de 128 bits. Estas direcciones se representan en formato hexadecimal y se dividen en ocho grupos de 16 bits, separados por dos puntos. Un ejemplo es 2001:0db8:85a3:0000:0000:8a2e:0370:7334. IPv6 proporciona un número casi ilimitado de direcciones, lo que resuelve el problema de agotamiento de direcciones en IPv4.

Máscaras de Subred

Las máscaras de subred son herramientas fundamentales para dividir una red en segmentos más pequeños, lo que facilita la administración y mejora el rendimiento. Funcionan al identificar qué parte de la dirección IP corresponde a la red y cuál a los dispositivos dentro de esa red:

  • Máscara de Subred IPv4: Una máscara de subred se representa también en formato decimal con puntos, como 255.255.255.0. En esta máscara, los primeros 24 bits (255.255.255) indican la red, y los últimos 8 bits (0) indican los dispositivos dentro de esa red. Esto permite que una red tenga hasta 254 dispositivos únicos.
  • Máscara de Subred IPv6: En IPv6, las máscaras de subred se representan con notación CIDR (Classless Inter-Domain Routing), como /64. Esto significa que los primeros 64 bits de la dirección son utilizados para la red y los restantes 64 bits para los dispositivos. IPv6 permite una mayor flexibilidad y granularidad en la asignación de direcciones.

Asignación de Direcciones IP

Las direcciones IP pueden ser asignadas de dos maneras principales, dependiendo de las necesidades de la red y la facilidad de administración:

  • Asignación Estática: En este método, la dirección IP se asigna manualmente a un dispositivo y no cambia a menos que se realice una modificación. Esto es ideal para dispositivos que necesitan una dirección fija, como servidores web, impresoras o dispositivos de red críticos. La asignación estática asegura que el dispositivo siempre tenga la misma dirección IP, lo que facilita la configuración de servicios y la gestión de red.
  • Asignación Dinámica: Utiliza un servidor DHCP (Dynamic Host Configuration Protocol) para asignar direcciones IP a los dispositivos de manera automática. Cuando un dispositivo se conecta a la red, el servidor DHCP le asigna una dirección IP disponible del rango definido. Este método simplifica la administración de red, evita conflictos de direcciones y es ideal para redes con un gran número de dispositivos que se conectan y desconectan frecuentemente, como redes domésticas o corporativas.

Protocolos de Comunicación

Los protocolos de comunicación definen las reglas y convenciones para la transmisión de datos en las redes. Son esenciales para garantizar que los datos se transmitan y reciban de manera correcta y eficiente. Cada protocolo cumple un papel específico en la red, facilitando diversas formas de comunicación y transferencia de datos.

TCP (Transmission Control Protocol)

TCP es uno de los protocolos más fundamentales en la comunicación de redes. Su principal función es proporcionar una entrega de datos que sea:

  • Confiable: Asegura que los datos lleguen a su destino sin errores. Si se detecta algún problema durante la transmisión, TCP solicita la retransmisión de los datos.
  • Ordenada: Garantiza que los datos se ensamblen en el orden correcto en el que fueron enviados, sin importar el orden en que fueron recibidos.
  • Verificada: Incluye mecanismos de verificación para comprobar la integridad de los datos durante el proceso de transmisión.

TCP es utilizado en aplicaciones donde la precisión y la entrega completa de los datos son críticas, como en la navegación web, correos electrónicos y la transferencia de archivos.

>

UDP (User Datagram Protocol)

UDP es otro protocolo importante, que se diferencia de TCP en varios aspectos clave:

  • Rápido: UDP ofrece una comunicación más rápida ya que no realiza el mismo tipo de verificación o control de errores que TCP.
  • Sin conexión: No establece una conexión previa entre el origen y el destino antes de transmitir datos, lo que reduce el tiempo de latencia.
  • Sin garantía de entrega: No asegura que los datos lleguen a su destino o que lleguen en el orden correcto. En cambio, se basa en la velocidad y eficiencia.

UDP es ideal para aplicaciones donde la velocidad es más importante que la precisión, como en transmisiones de video en tiempo real o juegos en línea.

HTTP/HTTPS

HTTP (Hypertext Transfer Protocol) y HTTPS (HTTP Secure) son protocolos utilizados para la transferencia de datos en la web:

  • HTTP: Es el protocolo básico para la transferencia de páginas web y otros recursos a través de la red. Funciona en la capa de aplicación y es la base para la navegación web.
  • HTTPS: Es la versión segura de HTTP, utilizando cifrado para proteger la integridad y la privacidad de los datos durante la transferencia. Esto es especialmente importante para la transmisión de información sensible, como en transacciones financieras o datos personales.

HTTPS asegura que las comunicaciones entre el navegador y el servidor estén protegidas contra ataques de intermediarios y espionaje.

>

FTP (File Transfer Protocol)

FTP es un protocolo utilizado para la transferencia de archivos entre sistemas a través de una red:

  • Transferencia de archivos: Permite la transferencia de archivos grandes de manera eficiente entre un cliente y un servidor.
  • Autenticación: Requiere autenticación mediante un nombre de usuario y una contraseña, aunque existen variantes seguras como FTPS y SFTP que añaden cifrado adicional.

FTP es comúnmente utilizado para la carga y descarga de archivos en servidores web y para la administración de archivos en sistemas remotos.

SMTP (Simple Mail Transfer Protocol)

SMTP es el protocolo estándar para el envío de correos electrónicos a través de Internet:

  • Envio de correos: Facilita la transferencia de mensajes de correo electrónico desde el cliente de correo electrónico al servidor de correo, y luego entre servidores hasta que llega al destino.
  • Función de relay: Puede actuar como un relay entre diferentes servidores de correo para asegurar que los mensajes lleguen a su destino final.

SMTP es fundamental para la comunicación por correo electrónico, trabajando en conjunto con otros protocolos como POP3 o IMAP para la recepción y gestión de mensajes.

Configuración y Administración

La configuración y administración de redes implica la instalación de dispositivos de red y asegurarse de que funcionen de manera eficiente. Esto incluye tareas como la configuración de direcciones IP, la monitorización de la red y el uso de herramientas de diagnóstico para resolver problemas de red.

  • Configuración de IP: Configuración de direcciones IP en varios dispositivos como laptops, switches Aruba, servidores, access points Aruba y dispositivos móviles. A continuación, se muestran ejemplos de configuraciones IP estáticas y dinámicas:
    • Laptops (Windows):
      • IP Estática: 
        1. Abre el "Panel de Control".
        2. Ve a "Centro de redes y recursos compartidos".
        3. Haz clic en "Cambiar configuración del adaptador".
        4. Haz clic derecho en tu conexión de red y selecciona "Propiedades".
        5. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" y haz clic en "Propiedades".
        6. Marca "Usar la siguiente dirección IP" y configura los campos:
           - Dirección IP: 192.168.1.100
           - Máscara de subred: 255.255.255.0
           - Puerta de enlace predeterminada: 192.168.1.1
           - Servidor DNS preferido: 8.8.8.8
           - Servidor DNS alternativo: 8.8.4.4
        7. Haz clic en "Aceptar" para guardar los cambios.
      • IP Dinámica (DHCP): 
        1. Abre el "Panel de Control".
        2. Ve a "Centro de redes y recursos compartidos".
        3. Haz clic en "Cambiar configuración del adaptador".
        4. Haz clic derecho en tu conexión de red y selecciona "Propiedades".
        5. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" y haz clic en "Propiedades".
        6. Marca "Obtener una dirección IP automáticamente" y "Obtener la dirección del servidor DNS automáticamente".
        7. Haz clic en "Aceptar" para guardar los cambios.
    • Switches Aruba:
      • IP Estática: 
        1. Accede a la interfaz web del switch Aruba.
        2. Ve a "Configuration" > "Interfaces".
        3. Selecciona la interfaz de administración.
        4. Configura los siguientes campos:
           - Dirección IP: 192.168.1.10
           - Máscara de subred: 255.255.255.0
           - Puerta de enlace predeterminada: 192.168.1.1
        5. Guarda la configuración.
      • IP Dinámica (DHCP): 
        1. Accede a la interfaz web del switch Aruba.
        2. Ve a "Configuration" > "Interfaces".
        3. Selecciona la interfaz de administración.
        4. Marca la opción "Obtener dirección IP automáticamente (DHCP)".
        5. Guarda la configuración.
    • Servidores (Windows Server):
      • IP Estática: 
        1. Abre "Administrador del Servidor" y selecciona "Administrador de red".
        2. Ve a "Ethernet", haz clic derecho y selecciona "Propiedades".
        3. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" y haz clic en "Propiedades".
        4. Ingresa los siguientes valores:
           - Dirección IP: 192.168.1.200
           - Máscara de subred: 255.255.255.0
           - Puerta de enlace predeterminada: 192.168.1.1
           - Servidor DNS preferido: 8.8.8.8
           - Servidor DNS alternativo: 8.8.4.4
        5. Haz clic en "Aceptar" para aplicar los cambios.
      • IP Dinámica (DHCP): 
        1. Abre "Administrador del Servidor" y selecciona "Administrador de red".
        2. Ve a "Ethernet", haz clic derecho y selecciona "Propiedades".
        3. Selecciona "Protocolo de Internet versión 4 (TCP/IPv4)" y haz clic en "Propiedades".
        4. Selecciona "Obtener una dirección IP automáticamente" y "Obtener la dirección del servidor DNS automáticamente".
        5. Guarda los cambios.
    • Access Points Aruba:
      • IP Estática: 
        1. Accede a la interfaz web del Access Point Aruba.
        2. Ve a "Configuration" > "Network" > "IP Settings".
        3. Selecciona "Static" y configura los campos:
           - Dirección IP: 192.168.1.30
           - Máscara de subred: 255.255.255.0
           - Puerta de enlace predeterminada: 192.168.1.1
           - Servidor DNS: 8.8.8.8
        4. Guarda la configuración.
      • IP Dinámica (DHCP): 
        1. Accede a la interfaz web del Access Point Aruba.
        2. Ve a "Configuration" > "Network" > "IP Settings".
        3. Selecciona "DHCP" para obtener una dirección IP automáticamente.
        4. Guarda la configuración.
    • Dispositivos Móviles (Android/iOS):
      • IP Estática (Android): 
        1. Ve a "Configuración" > "Red e Internet" > "Wi-Fi".
        2. Mantén presionada la red a la que estás conectado y selecciona "Modificar red".
        3. Marca "Opciones avanzadas".
        4. Cambia de "DHCP" a "Estática".
        5. Ingresa los siguientes valores:
           - Dirección IP: 192.168.1.50
           - Puerta de enlace: 192.168.1.1
           - Tamaño de la máscara de subred: 24
           - Servidor DNS 1: 8.8.8.8
           - Servidor DNS 2: 8.8.4.4
        6. Guarda los cambios.
      • IP Dinámica (DHCP) (iOS): 
        1. Ve a "Configuración" > "Wi-Fi".
        2. Toca el icono de información (i) junto a la red a la que estás conectado.
        3. En la sección "Configuración de IP", selecciona "DHCP".
        4. Asegúrate de que "Obtener dirección IP" esté activado.
        5. Sal de la configuración; los cambios se guardarán automáticamente.
  • Herramientas de Diagnóstico: Herramientas como `ping` y `traceroute` ayudan a resolver problemas de red. Por ejemplo, puedes usar el siguiente comando para hacer ping a un host:
    • ping www.ejemplo.com

Todas las configuraciones de red deben ser gestionadas y mantenidas por el departamento de OT, bajo la supervisión de Kaleb Chavira. Es crucial monitorear y actualizar constantemente las configuraciones de red para mantener la integridad y seguridad de la misma.

Cursos de Ciberseguridad de Fortinet

Fortinet ofrece una serie de cursos gratuitos diseñados para mejorar tu conocimiento en ciberseguridad. Estos cursos forman parte del programa Network Security Expert (NSE) y son accesibles tanto para principiantes como para aprendices de nivel intermedio.

  • NSE 1: Network Security Associate - Básico I

    Este curso de nivel inicial proporciona una comprensión fundamental del panorama de amenazas, los conceptos básicos de la ciberseguridad y los diversos tipos de productos y soluciones de seguridad en red. Ideal para quienes desean comenzar su camino en la ciberseguridad.

  • NSE 2: Network Security Associate - Básico II

    El curso NSE 2 amplía los conceptos introducidos en NSE 1, profundizando en las tecnologías de ciberseguridad y el papel que desempeñan en la protección de información y redes. Este curso es perfecto para aquellos que quieren ampliar su conocimiento básico.

  • NSE 3: Network Security Associate - Basico III

    NSE 3 ofrece un entendimiento a nivel intermedio de los productos y soluciones de Fortinet. El curso se centra en cómo estos productos protegen contra tipos específicos de amenazas. Es adecuado para aquellos que ya tienen una comprensión básica de ciberseguridad y desean aprender sobre el enfoque de Fortinet.
